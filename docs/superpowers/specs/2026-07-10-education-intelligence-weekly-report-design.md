# 教育行业情报双周报 Skill 重构设计

## 目标

将两个 Skill 从“按日期汇总新闻”改造成“从公开信息中筛选重要变化并形成研究判断”的教育行业情报工作流。最终报告供产业研究、投资讨论、内部汇报和项目机会发现使用，不能用低价值信息或检索结论凑页数。

沟通任务：读者应在十分钟内知道本期教育行业发生了哪些重要变化、这些变化为什么重要、对应哪些趋势和机会，以及下一期要验证什么。

## 基线诊断

目标样稿为 `教育行业观察20260622-0705.pdf`，失败样稿为 `教育行业观察_20260626-20260709_公开来源版.pptx`。

失败样稿暴露的问题：

- 候选信息主要来自教育部网站，没有覆盖融资、公司、行业媒体和地方政策模块。
- “新闻发言人名单”等低价值信息入选，说明只有字段完整性检查，没有研究价值门槛。
- 同一份规划拆成多页，造成重复信息占用正文。
- “未检索到事项”被做成正文页，最低事项数反而诱发凑数。
- 每页只有事件摘要和一句泛化影响，没有背景、商业模式、行业趋势和后续验证问题。
- 生成器固定为 16:9 文本页，目标样稿为 4:3，并需要支持表格等证据型版式。

## 方案选择

### 方案一：只加强提示词

改动小，但无法机械拦截低分事项、重复事项、字段缺失和虚假“已完成”状态。不能解决本次根因。

### 方案二：分层情报工作流

采用 `检索底稿 -> 评分筛选 -> 事件研究卡 -> 周报叙事 -> 自动验收 -> PPT`。判断规则写进 Skill，机械规则写进校验器和生成器。两个 Skill 共用质量模型，保留各自行业边界。此方案为本次采用方案。

### 方案三：独立多 Agent 与知识库流水线

长期可把采集、研究、周报和知识库拆成独立系统，但会引入任务调度、存储和增量更新，超出本次“先让 Skill 可用”的范围。

## 架构

### 1. 检索底稿

每个必跑模块都记录查询、来源、日期、候选数和空结果。候选事项至少包含标题、日期、主体、分类、关键事实、来源、缺失字段、入选或剔除理由。

### 2. 价值评分

每条候选事项按五个维度各评 1-5 分：行业影响、政策重要性、商业价值、技术相关性、投资相关性。

事项只有在事实和来源门槛通过后才能评分。满足以下任一条件方可入选：

- 总分不低于 16 分；
- 行业影响不低于 4 分，且其余四项中至少一项不低于 4 分。

每个维度必须写一句可由事实支撑的评分理由。不能为了达到页数提高评分。

### 3. 去重与组合

- 同一来源文件原则上最多形成一个正文事项。
- 同一主体、同一政策或同一交易的不同报道合并成一张事件研究卡。
- 同一栏目不得超过正文事项的一半；确因报告期事实高度集中时，在质量报告中说明。
- “未检索到”“搜索情况”“来源不足”等内容只进入检索底稿或缺口说明，不进入 PPT 正文。

### 4. 事件研究卡

每个入选事项必须回答：

- 发生了什么：时间、主体、动作、金额或关键条款。
- 主体是谁：业务、客户、产品或政策背景。
- 为什么重要：改变了什么行业变量，代表什么趋势。
- 谁受益或承压：公司、学校、用户、产业环节或商业模式。
- 下一步跟踪什么：一个可验证的问题、指标或时间点。

### 5. 周报叙事

报告结构固定为：

1. 封面。
2. 本期核心观点，最多 3 条，由入选事件归纳，不先写结论再找材料。
3. 重点事件，每页 1 个主事项。
4. 本期行业判断，100-200 字，说明共同趋势、机会、风险和下一期验证重点。

教育行业观察持续扫描 AI+教育；职教行业周报持续扫描政策、院校、产教融合和企业事项。若某模块无合格事项，只在检索底稿记录，不生成空白栏目页。

### 6. PPT 版式

- 默认 4:3，匹配目标样稿。
- 封面最简；正文保留栏目条、左侧短标题、右侧内容区。
- 每页 3-5 个事实或研究要点，另设“行业判断”和“后续跟踪”。
- 支持可选证据表格；没有必要时不强行加图表。
- 来源置于页脚并同时生成独立来源清单。
- 不显示提示词、自评过程、检索失败或制作说明。

## 停止与返工

出现以下任一情况不得生成正式 PPT：

- 合格正文事项少于 5 个。
- 任一事项没有可访问来源、日期、主体、事实、研究解释或跟踪问题。
- 重大交易、上市公司事项、政策或院校设置缺少对应的一手来源。
- 重复事项、低价值事项或检索结论被用于凑页数。
- 自动质量评分任一维度低于 8/10，或总分低于 32/40。

停止时交付检索底稿、候选池、剔除理由和缺口说明，不交付伪完整 PPT。

## 验收

### 内容验收

- 每个事件都同时包含事实、主体背景、行业解释和跟踪建议。
- 至少 5 个合格事项，且没有检索结论页。
- 核心观点能回指至少 2 个正文事项。
- 重要事项使用一手来源，媒体来源仅作补充。

### 结构验收

- 4:3 页面，封面、核心观点、事件页、本期判断完整。
- 正文每页 1 个事项，3-5 个要点，标题不溢出。
- 表格、来源和页码可读，无重叠、裁切或黑屏。

### 行为验收

- 用 0709 JSON 运行旧规则时应暴露失败；新校验器必须拒绝该 JSON。
- 合格样例应通过校验、生成 PPT 和来源清单。
- 两个 Skill 分别完成独立前向测试后才能同步运行时和 GitHub。

## 非本次范围

- 不接入邮箱或邮箱附件。
- 不建设 Obsidian 知识库和长期事件数据库。
- 不自动执行交易或投资决策。
- 不以复刻目标稿字体和颜色为主要目标，优先复现其研究密度和信息结构。
