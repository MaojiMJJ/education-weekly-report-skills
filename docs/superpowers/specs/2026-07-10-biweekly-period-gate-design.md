# 教育双周报报告期硬门槛设计

## 目标

教育行业观察必须追踪报告期内新发生、正式披露或正式生效的事项。来源文章在报告期内发布，不等于文章所总结的历史事项属于本期。任何栏目没有本期合格事项时直接省略，不用跨期内容补页。

## 问题定义

现有结构只有 `event_date` 和来源 `published_at`。生成者可以把一篇本期发布的半年盘点文章日期填入 `event_date`，从而绕过“日期在报告期内”的校验。自动校验只能看到日期格式正确，无法判断该日期究竟是事件日期还是文章发布日期。

## 采用方案

在现有事件卡中增加必填的 `period_trigger`，并由工作流、JSON 契约和自动校验共同约束：

```json
{
  "event_date": "2026-07-03",
  "period_trigger": {
    "type": "financing_announced",
    "description": "公司于7月3日宣布完成B轮融资",
    "source_url": "https://company.example/news/round-b"
  }
}
```

`event_date` 表示本期触发事件的实际日期；`period_trigger.source_url` 必须与 `sources` 中已核验的直接来源完全一致。

## 允许的本期触发类型

- `policy_issued`：政策、通知或正式决定发布。
- `policy_effective`：此前发布的政策在本期正式生效。
- `financing_announced`：融资完成或正式宣布。
- `transaction_signed`：并购、投资或重大合作正式签约、完成或公告。
- `filing_published`：上市公司、交易所或监管机构发布新增正式文件。
- `product_launched`：产品正式发布或上线，不含预热和媒体回顾。
- `deployment_started`：学校采购、中标、试点或规模化部署形成新增事实。
- `official_data_released`：官方或公司首次发布具有行业价值的新数据。
- `material_business_update`：其他可验证的重大业务进展。

“媒体盘点、半年回顾、季度总结、历史梳理、观点文章”不是合法触发类型。它们只能作为候选线索、背景资料或当前事件的补充来源。

## 分栏目日期规则

### 融资与交易

以融资宣布/完成、并购签约/完成、IPO申请递交/受理、正式上市日期为准。媒体在本期发布的历史融资汇总不能入选。融资栏目有本期事项时放在正文最前；没有则整个栏目省略。

### 政策与监管

以文件发布、正式决定或生效日期为准。对旧政策的本期解读、回顾和宣传不算新政策。旧文件在本期生效时可以入选，但必须由官方文件明确支持生效日期。

### 上市公司

以公告、交易所文件、业绩披露、签约或完成日期为准。例行宣传和对历史业务的媒体总结不算本期事项。

### AI产品与行业动态

以正式发布、上线、采购、中标、部署或新增经营数据披露日期为准。仅介绍既有产品功能的文章不算本期进展。

## 自动校验

生成器在现有校验基础上增加：

1. 每个事项必须包含完整 `period_trigger`。
2. `period_trigger.type` 必须属于允许列表。
3. `event_date` 必须在报告期内。
4. `period_trigger.source_url` 必须出现在该事项 `sources` 中，且来源已核验。
5. `period_trigger.description` 必须说明主体、日期和新增动作，不能写“文章发布”“盘点显示”等检索过程。
6. `policy_effective` 可以使用报告期前发布的官方来源；其他触发类型至少一个直接来源发布日期应在报告期内。
7. 缺少本期触发证据的事项拒绝生成PPT，不允许用高评分绕过。

## 回归测试

- 本期发布的“上半年52起融资盘点”，底层交易均在此前发生：拒绝。
- 本期媒体重新解读旧政策，没有新增发布或生效：拒绝。
- 公司本期正式宣布完成融资，并有公司或投资方直接来源：通过。
- 旧政策在本期正式生效，官方文件明确生效日期：通过。
- 融资栏目为空但报告总事项达到门槛：通过，PPT不生成空栏目页。
- `period_trigger.source_url` 不在来源清单或来源未核验：拒绝。
