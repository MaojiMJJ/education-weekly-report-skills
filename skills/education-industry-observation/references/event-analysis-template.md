```text

事件类型：
本期触发：

公开项目符号：
1.
2.
3.
4.

内部分析（不得进入 PPT）：
内部跟踪（不得进入 PPT）：
直接来源：
```

## 写作顺序

1. 第一条写报告期内谁在什么时间正式做了什么，包含金额、股比、文件名或关键动作。
2. 后续条目补充主体背景、产品或政策内容、客户、学校、用户、实施方式、条款或规模。
3. 只保留可由来源直接支持的事实，不在公开要点中加入泛化“行业判断”标签。
4. 按 `report-template.md` 的动态分页容量压缩，先删重复背景和宣传性形容词。

## 判断约束

- `analysis` 写“事实变化—影响机制—行业结果”，只用于内部。
- `tracking` 写可验证指标或节点，如预算、中标、用户、收入、交割或实施时间，只用于内部。
- 来源未披露的信息不得推算或补造。
- 公开项目符号不得出现“值得关注”“意义重大”“后续跟踪”“下一期验证”等套话或内部口径。
