# 信息源和检索流程

## 资料边界

- 公开来源：政府、交易所、公司、投资方、学校、采购平台和行业媒体公开网页。
- 用户文件：用户明确提供的 Word、PDF、Excel 或其他材料。
- 邮箱材料：只有用户明确允许时才能使用。用户禁止邮箱时，不读取、不摘要、不引用。

三类资料必须在底稿中分开标注。不能把搜索摘要或邮件标题当作事实来源。

## 来源优先级

1. 一手来源：政府文件、交易所公告、公司公告、学校官网、采购公告、投资方或被投公司原文。
2. 可靠行业媒体：多知网、芥末堆、蓝鲸财经教育频道、多鲸资本、黑板洞察。
3. 财经和地方媒体：用于补充背景或提供线索。
4. 聚合页、转载、自媒体和论坛：只作线索，不作最终证据。

每个正式事项都必须回到一手来源或原创直接报道。媒体转载只能补充线索；媒体与一手来源冲突时，以一手来源为准并记录差异。

## 必跑模块

每期必须全部执行并记录查询、日期、候选数和空结果。

### 政策与监管

- 教育部、国务院、省市教育厅和教育局。
- 人工智能教育、教育数字化、培训监管、招生制度、学校采购和教育消费。
- 逐项提取发文机关、政策名称、发布日期、生效时间和执行范围，明确是全国性还是地方性。
- 核心条款按禁止性规定、限制性要求、倡导性或支持性内容分类；某类没有条款时记录为空，不跨类拼凑。
- 重点追踪政策是否包含实施时间、预算、试点名单、课程学时、采购或考核要求。

### 上市公司

先读取 `listed-companies.md`，逐家公司检索报告期公告：收购、再融资、投资、合作、新产品、AI教育、业绩说明和业务调整。

### 融资与交易

检索 AI 学习、AI 教师、AI 测评、校园智能体、学习硬件、教育机器人、教育信息化和学校服务公司的融资、并购与战略投资。

只收录报告期内正式宣布或完成的融资、并购、收购、IPO递交、受理或上市事项。媒体在本期发布的上半年盘点、季度汇总和历史融资清单只作线索或背景，不能作为正文事项。

融资候选必须逐项查找并记录：

- 投资规模、轮次、币种、投资方和资金用途。
- 被投公司的业务定位、业务领域、B端或C端、业务板块、线上或线下、硬件或服务、成人或少儿用户。
- 成立时间、用户数、门店数、直营或加盟、流水、此前知名投资方。
- 营收规模和利润。未公开的数据显式写“未披露”，不得从估值、融资额或门店数推算。

### 业务合作与新设主体

- 检索培训机构、教育信息化公司、学校和高校的新业务、新合作、新学校筹备或正式设立。
- 核验合作各方、主体类型、合作或设立动作、业务或学校名称、区域、实施计划和商业化路径。
- 只有框架协议、活动签约且没有新增业务内容、执行节点或设立动作时不入选。

### AI+教育产品与落地

检索新产品时必须继续验证客户、付费方、学校数、用户数、采购、中标、续费或收入。只有发布会和功能描述的产品不入选。

### 行业媒体

逐站检索多知网、芥末堆、蓝鲸财经教育频道、多鲸资本和黑板洞察，不只依赖通用搜索结果。

## 检索底稿

每个查询记录：

| 字段 | 要求 |
| --- | --- |
| 模块 | 政策、上市公司、融资、AI+教育、行业媒体等 |
| 查询 | 实际使用的关键词或站内检索式 |
| 执行日期 | YYYY-MM-DD |
| 结果数 | 可进一步核验的候选数量 |
| 代表来源 | 一手来源优先 |
| 空结果说明 | 没有合格事项时记录，不进入 PPT |

## 候选池

每条候选记录：标题、分类、来源发布日期、本期触发类型、事件日期、触发动作、主体、关键事实、直接来源名称和URL、来源层级、访问核验状态、核验日期、已披露字段、缺失字段、初步评分、入选或剔除理由。

候选池通常应明显大于最终事项数。若检索后只有 5 条候选，不要直接出刊，应检查是否遗漏地方政策、公司公告、融资和行业媒体模块。

## 速览与深析分流

候选通过报告期、直接来源和去重门槛后，再按信息价值和可展开程度分流：

- `content_role: event`：高价值深析事项，必须满足五维评分门槛，写清主体背景、事实、影响机制、受益方和风险。
- `content_role: brief`：本期确有新增、对行业判断有补充价值，但尚不适合独立深析一页的事项；保留 2-3 条可核验事实和“为什么重要”。
- 速览也必须有明确 `event_date`、`period_trigger` 和已打开的直接来源。不得把“近日”、搜索摘要、跨期旧闻、盘点或无来源消息放入速览。
- 同一事项只能进入一层；若速览与深析重复，保留深析并删除重复速览。
- 双层模式通常至少覆盖 3 个栏目、5 个速览和 5 个深析，总事项不少于 10 个。

## 核验顺序

1. 识别本期新增动作，不先看文章发布日期：发生、正式披露、签约、上市、发布、生效、中标或部署。
2. 填写 `period_trigger`，确认 `event_date` 在报告期内，并找到直接支持该动作的来源。
3. 确认主体和动作，不采用只有标题、搜索摘要、盘点、回顾或转述的内容。
4. 找到一手来源；媒体只可补充公司背景、行业数据和采访信息，原创直接融资报道必须明确对应本期单笔交易。
5. 实际打开 URL，记录 `access_status: verified` 和 `access_checked_at`；核验日期不得早于发布日期，搜索摘要、保留域名和占位链接不算核验。
6. 补齐金额、规模、条款、客户、产品和业务进展。
7. 合并同一事项的多条来源，保留事实差异。
8. 通过报告期和事实门槛后再评分，低分事项写明剔除理由。

## 覆盖检查

- 是否跑完全部模块，而不是只看教育部门户。
- 是否至少覆盖政府、公司或学校、行业媒体等不同来源层级。
- 是否出现同一域名或同一文件支撑多页正文。
- 是否有栏目缺口；缺口应留在底稿，不应用弱事项填充。
- 某模块没有报告期内合格事项时是否直接省略，而不是用跨期内容或“本期无新增”页面补位。
