# 信息源和检索流程

## 资料边界

- 公开来源：政府、交易所、公司、投资方、学校、采购平台和行业媒体公开网页。
- 用户文件：用户明确提供的 Word、PDF、Excel 或其他材料。
- 邮箱材料：只有用户明确允许时才能使用。用户禁止邮箱时，不读取、不摘要、不引用。

三类资料必须在底稿中分开标注。不能把搜索摘要或邮件标题当作事实来源。

## 来源优先级

1. 一手来源：政府文件、教育主管部门公示、学校官网、交易所和公司公告、采购公告、投资方或被投公司原文。
2. 可靠行业媒体：多知网、芥末堆、蓝鲸财经教育频道、多鲸资本、黑板洞察。
3. 财经和地方媒体：用于补充背景或提供线索。
4. 聚合页、转载、自媒体和论坛：只作线索，不作最终证据。

政策、院校设置、上市公司、重大合作和交易必须回到一手来源。媒体与一手来源冲突时，以一手来源为准并记录差异。

## 必跑模块

每期必须全部执行并记录查询、日期、候选数和空结果。

### 政策

- 教育部、国务院、人社部、国家发展改革委及省市教育厅和教育局。
- 新双高、职业本科、职普融通、产教融合、专业目录、实训基地、技能培训和就业政策。
- 重点验证资金、试点、学校名单、申报、验收和实施时间。

### 职业本科与院校

- 职业技术大学设置、职业学院升本、院校合并或停止办学。
- 职业本科专业、招生计划、新设二级学院和管理体制。

### 产业学院与校企合作

- 产业学院、订单班、现场工程师、现代学徒制、市域产教联合体和行业产教融合共同体。
- 必须继续查投入、设备、课程、师资、招生、岗位和就业，只有揭牌不入选。

### 上市公司

先读取 `listed-companies.md`，逐家公司检索报告期公告：并购、投资、合作、合同、新业务、新产品、业绩说明和职教业务调整。

### 非上市企业与融资

检索职业培训、技能评价、产教融合平台、实训设备、职教信息化和企业培训公司的融资、并购与产品落地。

### 就业与技能

检索职业标准、技能等级、补贴性培训、就业见习和园区企业用工。泛就业新闻只有在包含职教学校、技能培训或企业岗位机制时入选。

### 行业媒体

逐站检索多知网、芥末堆、蓝鲸财经教育频道、多鲸资本和黑板洞察，不只依赖通用搜索结果。

## 检索底稿

每个查询记录：

| 字段 | 要求 |
| --- | --- |
| 模块 | 政策、院校、产教融合、上市公司、融资、就业技能等 |
| 查询 | 实际使用的关键词或站内检索式 |
| 执行日期 | YYYY-MM-DD |
| 结果数 | 可进一步核验的候选数量 |
| 代表来源 | 一手来源优先 |
| 空结果说明 | 没有合格事项时记录，不进入 PPT |

## 候选池

每条候选记录：标题、分类、发布日期、主体、关键事实、来源名称、URL、来源层级、访问核验状态、核验日期、已披露字段、缺失字段、初步评分、入选或剔除理由。

候选池通常应明显大于最终事项数。若检索后只有 5 条候选，不要直接出刊，应检查是否遗漏地方政策、院校官网、公司公告、融资和行业媒体模块。

## 核验顺序

1. 确认日期在报告期内，或报告期内存在关键新增披露。
2. 确认主体和动作，不采用只有标题、搜索摘要或转述的内容。
3. 找到一手来源；媒体可补充企业和院校背景。
4. 实际打开 URL，记录 `access_status: verified` 和 `access_checked_at`；搜索摘要不算核验。
5. 补齐金额、规模、专业、课程、招生、投入、岗位和业务进展。
6. 合并同一事项的多条来源，保留事实差异。
7. 通过事实门槛后再评分，低分事项写明剔除理由。

## 覆盖检查

- 是否跑完全部模块，而不是只看教育部门户。
- 是否同时覆盖政策、学校或企业、行业媒体等不同来源层级。
- 是否出现同一文件支撑多页正文。
- 是否把挂牌和活动误当成产教融合成果。
- 栏目缺口应留在底稿，不应用弱事项填充。
